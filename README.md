# 포스코센터 근처 점심·저녁 추천 식당 가이드

포스코센터(서울 강남구 테헤란로 440) 근처에서 **점심·저녁 식사 장소를 상황별로 빠르게 비교·선택**하기 위한 사내 공유용 정적 웹페이지입니다.
단순 맛집 소개가 아니라 **팀 점심 / 임원 동석 / 외부 손님 접대 / 회식 / 조용한 대화 / 빠른 식사 / 가성비 / 룸 / 예약 가능 여부** 등 업무 상황에 맞춰 필터링하도록 설계했습니다.

---

## 1. 프로젝트 목적

- 포스코센터 도보권(우선 15분 이내) 식당을 **정보 출처(네이버지도·카카오맵) 기준**으로 정리
- 예약·룸·접대 적합성 등 **업무적으로 판단 가능한 정보** 중심 정리 (감성적 맛집 표현 지양)
- 카드/표/필터/검색/즐겨찾기로 **3초 안에 후보를 좁힐 수 있는** 사내 도구 제공

---

## 2. 파일 구조

```
posco-center-restaurant-guide/
├─ index.html          # 화면 구조
├─ style.css           # 스타일 (화이트/라이트그레이 + 포스코 블루)
├─ script.js           # 필터·검색·정렬·즐겨찾기·CSV·랜덤 등 로직 (Vanilla JS)
├─ restaurants.json    # 식당 데이터 (단일 원본)
├─ README.md           # 본 문서
└─ _server.js          # (선택) python이 없을 때 쓰는 Node 정적 서버
```

`_server.js`는 운영 필수 파일이 아니며, python이 없는 환경에서의 로컬 실행 편의용입니다.

---

## 3. 로컬 실행 방법

`index.html`을 파일로 직접 열면 브라우저 보안 정책(`file://`에서의 `fetch` 제한) 때문에 `restaurants.json`을 못 읽을 수 있습니다.
**반드시 로컬 서버로 실행**하세요.

### 방법 A. Python (설치되어 있는 경우)
```bash
cd posco-center-restaurant-guide
python -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

### 방법 B. Node.js
```bash
cd posco-center-restaurant-guide
node _server.js
# http://localhost:8765 접속
```
또는 `npx serve .` / `npx http-server .` 등도 가능합니다.

### 방법 C. VS Code
`Live Server` 확장에서 `index.html` → **Open with Live Server**.

> 로딩에 실패하면 화면에 안내 문구가 표시되며, 대부분 “서버 없이 파일로 열었을 때” 발생합니다.

---

## 4. 식당 데이터 추가 / 수정 방법

모든 데이터는 **`restaurants.json` 한 파일**에만 있습니다. 이 파일만 수정하면 화면·필터·요약이 자동 반영됩니다.

1. `restaurants.json`을 텍스트 에디터로 엽니다.
2. 배열에 객체를 추가하거나 기존 객체를 수정합니다.
3. `id`는 `restaurant-061`처럼 **중복 없이 순차** 부여합니다. (현재 `restaurant-001`~`restaurant-060`까지 사용 중)
4. 저장 후 브라우저를 새로고침합니다.

> JSON은 마지막 항목 뒤에 쉼표(`,`)가 있으면 깨집니다. 저장 후 로딩이 안 되면 콤마·따옴표를 먼저 확인하세요.

## 5. JSON 필드 설명

| 필드 | 설명 | 허용값 / 형식 |
| --- | --- | --- |
| `id` | 고유 ID | `restaurant-001` 형식 |
| `name` | 식당명 | 문자열 |
| `category` | 음식 종류(필터 기준) | 한식 / 중식 / 일식 / 양식 / 고기·구이 / 해산물 / 분식·가벼운 식사 / 카페·디저트 / 기타 |
| `subType` | 세부 분류 | 예: 한정식, 이자카야 |
| `mealTime` | 식사 시간 | `["lunch","dinner"]` 배열 |
| `representativeMenus` | 대표 메뉴 | 문자열 배열 |
| `lunchPriceRange` / `dinnerPriceRange` | 가격대 | `1만원대` / `2만원대` / `3만원대` / `5만원 이상` / `확인 필요` |
| `distanceMinutes` | 도보 분(정렬·거리필터) | 숫자 |
| `distanceLabel` | 표시용 거리 | 예: `도보 5분` |
| `address` / `phone` | 주소 / 전화 | 문자열 (`확인 필요` 허용) |
| `businessHours` / `breakTime` / `closedDays` | 영업시간 / 브레이크 / 휴무 | 문자열 (`확인 필요` 허용) |
| `lunchMenu` / `dinnerMenu` | 점심·저녁 메뉴 상세 | 문자열 (`확인 필요` 허용) |
| `room` | 룸 정보 | `{status: 있음/없음/확인 필요, capacity, note}` |
| `reservation` | 예약 정보 | `{status: 예약 가능/예약 권장/예약 불가/확인 필요, method, note}` |
| `parking` | 주차 | `{status: 가능/불가/확인 필요, note}` |
| `recommendedFor` | 추천 상황(필터) | 팀 점심 / 임원 동석 / 외부 손님 접대 / 회식 / 조용한 대화 / 빠른 식사 / 가성비 |
| `atmosphere` | 분위기 | 문자열 배열 (`분위기 좋음` 포함 시 “분위기 좋은 곳” 필터 대상) |
| `tags` | 카드 표시 태그 | 문자열 배열 |
| `comment` | 업무용 한줄 코멘트 | 건조·실용적으로 |
| `cautions` | 주의/확인 필요 사항 | 문자열 |
| `mapLinks` | 지도 링크 | `{naver, kakao, google}` (검색 딥링크) |
| `sourceLinks` | 참고 링크 | 공식/지도/캐치테이블 URL 배열 |
| `lastVerified` | 마지막 확인일 | `YYYY-MM-DD` |
| `priority` | 추천순 정렬 우선순위 | 숫자(작을수록 우선) |

## 6. 실제 데이터 입력 시 주의사항 (중요)

이 가이드의 데이터는 **네이버지도·카카오맵의 공개 정보(지역검색) 기준**으로 수집했습니다. 다음 원칙을 반드시 지켜 주세요.

- **확인되지 않은 정보는 추측해서 채우지 않습니다.** 미확인 항목은 `확인 필요`로 둡니다.
- **룸 유무·예약 가능 여부·영업시간·브레이크타임·휴무일**은 변동성이 커서, 확인되지 않았으면 `확인 필요`로 표기되어 있습니다.
- **가격대**는 메뉴판으로 직접 확인한 값이 아니면 “공개 리뷰·업종 기준 추정치”이며, 각 카드 `cautions`에 명시했습니다. **실제 예산 산정 전 메뉴판 확인**이 필요합니다.
- **전화번호**는 지역검색 API에서 값이 비어 있어 현재 전부 `확인 필요`입니다. 실제 예약을 위해선 지도 앱에서 전화번호를 확인해 채워 주세요.
- **예약/룸** 관련은 실제 전화 확인 전까지 단정하지 않고 `예약 가능`(캐치테이블 등 예약 채널 확인됨) / `예약 권장` / `확인 필요`로 신중히 표기했습니다.
- 네이버지도와 카카오맵 정보가 다르면 `cautions`에 “지도별 정보 상이, 방문 전 확인 필요”로 적어 주세요.

## 7. 네이버지도 / 카카오맵 기준 정보 확인 방법

각 카드 하단의 **네이버 / 카카오 / 구글** 버튼은 식당명 검색 딥링크입니다. 확인 순서 권장:

1. **네이버지도** — 영업시간·브레이크·휴무·전화·메뉴·예약(네이버 예약) 우선 확인
2. **카카오맵** — 룸·주차·전화 교차 확인
3. **공식 홈페이지 / 캐치테이블 / 공식 SNS** — 코스·예약 가능 여부 확인 (`sourceLinks` 참고)
4. **블로그/리뷰** — 보조 참고만 (분위기·대기 등)

확인 후에는 해당 필드를 채우고 **`lastVerified`(확인일)를 갱신**하세요.

---

## 8. `restaurants.json` 업데이트 규칙

- **단일 원본 유지:** 데이터는 `restaurants.json`에만 둡니다.
- **확인일 관리 필수:** 값을 확인/수정할 때마다 그 식당의 `lastVerified`를 갱신합니다. 예약·룸·영업시간은 수시로 바뀌므로 **확인일이 곧 신뢰도**입니다.
- **분기 1회 이상 재검증** 권장. 특히 접대/회식용(임원 동석·외부 손님 접대) 식당은 사용 직전 재확인.
- 폐업/이전 확인 시 해당 객체를 삭제하거나 `cautions`에 상태를 명시합니다.
- 필터가 정상 동작하려면 `category`·`recommendedFor`·`room.status`·`reservation.status`·가격대 값이 **위 표의 허용값과 정확히 일치**해야 합니다.

## 9. 주요 기능 요약

- 요약 카드(전체/룸/예약/점심·저녁 추천 수)
- 상황별 원클릭 추천 버튼 8종
- 복수 선택 필터(식사시간·종류·목적·거리·룸·예약·가격) + 검색 + 정렬 6종

- 즐겨찾기(localStorage) 및 즐겨찾기만 보기
- 식당 정보 복사, 점심/저녁 랜덤 뽑기, CSV 내보내기, 인쇄용 보기
- 결과 개수·빈 결과·로딩 실패 안내, 접근성(aria-label·키보드·색상 대비) 고려
